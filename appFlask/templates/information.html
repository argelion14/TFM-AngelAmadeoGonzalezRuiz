    TFM

    Traditional access control models rely on roles (Role-Based Access Control, RBAC), which often lack flexibility and granularity. A more modern approach is Attribute-Based Access Control (ABAC), where access rights are granted based on user attributes like role, department, project, or operation type.

    ORI is a prototype authentication and access control server developed for a Master's Thesis. It issues access control tokens based on user attributes. The system includes authentication, token issuance, user/role/permission management, and a dynamic ruleset for DDS (Data Distribution Service) environments.

    • User Management: Create, edit and delete users. Assign superuser roles.
    • Certificate Management: Store and display public certificates per user.
    • Role Templates: Define role-based permissions with expiration and grant rules.
    • Access Rules: Define allow/deny rules for topics/domains with publish/subscribe actions.
    • Token Generation: Token-based access using cryptographically signed permissions.
    • Dashboard: Visualize database statistics and system usage.
